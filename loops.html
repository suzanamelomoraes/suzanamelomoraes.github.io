Control flow and Loops

Control flow is in computer science the order that the instructions or statements or functions are executed. In javascript, we read the code starting from the first line till the last line unless of course in the code there is some instructions or statements that changes that control flow.

The statements inside your source files are generally executed from top to bottom, in the order that they appear. Control flow statements, however, break up the flow of execution by employing decision making, looping, and branching, enabling your program to conditionally execute particular blocks of code.

Loops are a big part of control flow statement. They are the primary mechanism for telling a computer to perform the same task over and over again until a set of criteria are met.

All programming languages need ways of doing similar things many times, this is called iteration. Loops allow a block of statements to be executed repeatedly without actually writing them down numerous times. This is where for, while and do ... while loops are of use.

In a loop structure, the loop asks a question. If the answer requires action, it is executed. The same question is asked again and again until no further action is required. Each time the question is asked is called an iteration.
